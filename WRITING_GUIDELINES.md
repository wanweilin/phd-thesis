# 博士论文写作指导方针

## 一、总体原则

### 1.1 整合而非拼接
三篇小论文不是简单地首尾相连，而是要在统一的研究叙事下重新组织：
- 每章开头需说明本章与整体主题的关系
- 各章之间需要明确的过渡段落，阐明逻辑递进关系
- 总体绪论和结论章节必须原创撰写，不得照搬小论文的introduction/conclusion

### 1.2 构建统一叙事
论文的核心叙事应当贯穿始终：
> 数据是深度学习的基础，而**数据选择**——在训练、微调与推理的不同阶段，识别和利用最相关的数据——是提升模型效率与可靠性的关键。本论文系统研究深度神经网络全流程中的数据选择问题：在训练阶段，如何在计算预算约束下选取最优训练子集；在训练与压缩协同时，数据选择与模型剪枝如何相互增益；在推理阶段，如何判断输入样本是否处于模型的有效数据域内。

**关键：每篇论文需在"数据选择"这一主线下重新定位自身**

| 论文 | 在数据选择主线中的定位 | 需要在改编时强调的角度 |
|------|----------------------|----------------------|
| 论文二 (CADS) | 训练数据选择的基础问题 | 预算约束如何改变最优数据分布 |
| 论文三 (SWaST) | 数据选择与模型选择的耦合 | 两类"选择"相互依赖、协同优化 |
| 论文一 (NAP) | 推理时的数据相关性判断 | OOD检测本质是判断"此数据是否在训练分布内" |

## 二、各部分写作要点

### 2.1 第一章：绪论
- **研究背景**：从数据在深度学习中的核心地位出发，指出数据选择在不同阶段（训练/微调/推理）面临的独特挑战
- **问题陈述**：
  1. 训练阶段：全量数据代价高昂，如何在有限预算内选出最优子集？
  2. 训练与压缩协同阶段：数据选择与模型剪枝相互影响，如何联合优化？
  3. 推理阶段：测试样本来源未知，如何判断样本是否在模型的有效数据域内？
- **贡献列表**：以编号列表形式列出全部三篇工作，明确每篇工作回答了哪一阶段的数据选择问题
- **研究地图**：建议绘制一张示意图，展示三个工作在深度学习流程（数据→训练→模型→推理）中的位置
- **篇章结构**：用一段话介绍每章内容

### 2.2 第二章：相关工作
- 将三篇小论文中分散的related work统一整理，去除重复，补充交叉领域综述
- **以"数据选择"为主线组织综述结构**：
  - 训练数据选择与coreset方法（核心，直接支撑论文二、三）
  - 模型剪枝与计算效率（支撑论文三）
  - 双层优化（支撑论文二）
  - OOD检测（支撑论文一，需在最后说明其与数据选择的关联）
- **重点补充**：现有工作如何将数据选择局限在各自阶段，本文如何突破这一局限
- 建议用一段话明确说明：为何OOD检测可被理解为推理阶段的数据相关性判断

### 2.3 核心章节（第三、四、五章）
改写小论文时需要注意：
- **删去**各小论文自己的 related work 节（已统一到第二章）
- **扩展**实验分析部分，可补充跨论文的对比与消融实验
- **统一**符号系统：若三篇论文使用了不同的变量命名，需在论文中统一
- **补充**章节引言（1-2段）：在"数据选择"主线下说明本章所解决的子问题，以及与前后章的关系
- **补充**章节小结：总结本章贡献，并自然过渡到下一章

各章改写的重点关注：

**第三章（论文二，CADS）**
- 引言需强调：现有数据选择方法为何忽视预算导致失效，预算约束如何从根本上改变"什么数据是有用的"
- 与下一章（SWaST）的过渡：CADS解决了"在预算下选数据"，但未考虑模型结构本身也需要优化

**第四章（论文三，SWaST）**
- 引言需强调：数据选择与模型剪枝在现有工作中相互独立，本章揭示二者存在深层耦合
- 与上一章（CADS）的联系：同属训练阶段的数据相关���策，但本章将"选择"的维度从数据扩展到模型
- 与下一章（NAP）的过渡：前两章解决训练阶段问题，但训练好的模型部署时仍面临"输入数据是否有效"的判断问题

**第五章（论文一，NAP）**
- 引言需强调：模型训练完成后，推理阶段的数据相关性判断（OOD检测）是数据选择问题在部署侧的自然延伸
- 需明确建���与前两章的逻辑联系：训练数据选择决定了模型的有效域，OOD检测则是在推理时对该有效域的在线判断

### 2.4 第六章：总结与展望
- **工作总结**：每篇工作各一段，在"数据选择"主线下简洁总结各章贡献
- **统一视角**：从"深度学习全流程数据选择"的视角回顾三个工作，阐明它们共同构成了一个从训练到推理的系统性研究框架
- **局限性**：诚实列出每个工作的局限（通常小论文中已有部分）
- **未来工作**：围绕数据选择主线提出延伸方向，例如：
  - 训练与推理阶段数据选择的联合建模
  - 大模型（LLM）微调场景下的数据选择新挑战
  - 数据选择与模型架构搜索（NAS）的协同

## 三、技术写作规范

### 3.1 语言与格式
- 论文语言：**中文**（除非导师明确要求英文）
- 中文论文中的技术术语首次出现时需附英文原文，例如：分布外检测（Out-of-Distribution Detection, OOD）
- 公式编号采用全文统一编号，如 (3-1), (4-1) 等，不沿用小论文内的编号

### 3.2 图表规范
- 所有图表需重新制作或优化，确保分辨率和风格统一
- 图标题使用"图X-Y 标题"格式（章号-图序号）
- 每张图必须在正文中有引用，且引用在图之前出现
- 保留小论文中最关键的图，补充必要的对比图（如三个方法的统一对比图）

### 3.3 参考文献
- 统一使用一个.bib文件（合并三篇论文的bib，去除重复条目）
- 引用格式遵循所在院校/学院的毕业论文模板要求
- 自引时需注意：引用自己已发表的小论文是标准做法，无需回避

## 四、写作流程建议

```
阶段一：准备（约1-2周）
  □ 确定学校毕业论文LaTeX/Word模板
  □ 通读三篇论文，梳理符号系统差异
  □ 确定统一叙事框架（与导师对齐）
  □ 搭建完整目录结构

阶段二：起草（约4-6周）
  □ 撰写第一章绪论（全新，需包含"研究地图"示意图）
  □ 整合撰写第二章相关工作（全新，按数据选择主线组织）
  □ 改编第三章（论文二 CADS，训练数据选择）
  □ 改编第四章（论文三 SWaST，数据×模型联合选择）
  □ 改编第五章（论文一 NAP，推理时数据相关性判断）
  □ 撰写第六章总结（全新）

阶段三：整合与修改（约2-3周）
  □ 统一全文符号和术语
  □ 检查各章过渡是否自然
  □ 统一图表格式和编号
  □ 合并参考文献
  □ 全文润色

阶段四：审阅（约1-2周）
  □ 导师审阅
  □ 根据反馈修改
  □ 格式检查
  □ 提交盲审
```

## 五、利用Claude辅助写作的注意事项

- **提供原文**：让Claude处理具体章节时，始终提供该小论文对应章节的原始英文内容
- **分段处理**：一次只处理一个章节或一个小节，避免上下文丢失
- **明确任务**：区分"翻译"、"改写"、"扩写"、"整合"等不同任务类型
- **保留学术风格**：Claude生成的内容需经过审查，确保符合你的写作风格和学术规范
- **核实事实**：所有实验数字、引用内容需与原论文核对，不可依赖Claude的记忆

## 六、常见问题提示

| 问题 | 建议处理方式 |
|------|-------------|
| 三篇论文联系不够紧密 | 在绪论中构建"研究地图"，用一张图展示三者关系 |
| 符号系统冲突 | 以其中一篇为基准，其余论文改写时适配 |
| 实验部分过长 | 主体保留核心实验，其余移入附录 |
| 查重率过高 | 重写关键段落，尤其是方法描述部分（即使是自己的论文也需改写） |
| 字数不足 | 扩展实验分析、补充统一的对比实验、深化相关工作综述 |
