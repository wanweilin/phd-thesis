## 章节引言

**段落1 — 领域概览与组织逻辑**
- 本章围绕数据选择这一核心主题，从四个方面综述相关领域：训练数据选择方法（2.1）为第三、四章提供数据侧背景；模型冗余与效率优化（2.2）为第四章提供模型侧背景；数据选择中的优化方法（2.3）为第三章提供方法论工具；分布外检测（2.4）为第五章提供推理侧背景。
- 强调组织原则：不是罗列四个独立领域，而是始终围绕"数据选择与关键因素的交互"这一视角串联。

**段落2 — 共性不足的预告**
- 这四个领域各自已有丰富的研究积累，但一个共性问题是：现有工作倾向于将数据选择作为独立决策研究，较少关注数据选择与计算预算、模型容量、推理时分布偏移等因素的交互影响。
- 本章在综述各领域时，会在每节末尾指明这一不足，为后续三章的研究动机提供铺垫。

## 2.1 训练数据选择方法：核心子集选择与数据混合

**段落1 — 核心子集（Coreset）选择方法**
- 基于优化的方法：子模函数最大化保证近似最优性（如 Facility Location），梯度匹配方法（GradMatch）通过匹配全集梯度来选择子集，双层优化方法（PBCS 等）以验证集性能为目标优化子集。
- 基于训练动态的启发式方法：遗忘事件（Forgetting Events）记录训练中被反复遗忘的样本，EL2N/GraNd 利用早期训练信号估计样本难度，这些方法计算高效但缺乏理论保障。
- 基于几何/距离的方法：Herding、K-Center-Greedy 等基于特征空间距离选择代表性样本。

**段落2 — 大模型微调中的数据选择**
- LLM 微调场景的特殊性：数据来源多样（人工标��、模型生成、开源数据集），质量参差不齐，且单次训练成本极高，使得数据选择尤为关键。
- 代表性方法：LIMA 证明少量高质量数据即可有效微调，DEITA 综合质量、复杂度和多样性三个维度打分排序，BlendSearch 等利用自然语言指标筛选。
- 与传统 coreset 选择的区别：LLM 数据选择更依赖启发式质量指标而非梯度信息，且常需处理来源级（source-level）的混合配比问题。

**段落3 — 数据集蒸馏与课程学习**
- 数据集蒸馏（Dataset Distillation）：通过优化生成少量合成样本来压缩数据集信息，与 coreset 选择的"选"相对，蒸馏是"造"。优点是极致压缩，但合成样本缺乏可解释性且生成过程计算昂贵。
- 课程学习（Curriculum Learning）：按难度顺序组织训练样本，关注的是训练顺序而非子集选择。与 coreset 选择有交叉（都涉及样本重要性评估），但目标不同。
- 本文聚焦于从现有数据中选择子集的问题，蒸馏和课程学习不是本文的直接研究对象，但为理解数据效率的全景提供参照。

**段落4 — 共性不足：忽视计算预算的交互**
- 上述所有方法在选择数据时，隐含假设训练会运行到收敛或至少运行足够长时间，最优子集与实际可用的计算预算无关。
- 但实际中计算预算差异巨大（从小规模实验到大规模训练）。已有研究发现，精心设计的选择策略在某些预算下��至不如随机选择——暗示最优子集确实依赖于预算。
- 这一不足正是第三章 CADS 要解决的核心问题：将计算预算作为数据选择的显式变量。

## 2.2 模型冗余与效率优化

**段落1 — 结构化剪枝**
- 结构化剪枝移除整个滤波器/通道，产生的稀疏模型可直接在标准硬件上加速。
- 代表性准则：基于 L1/L2 范数的滤波器重要性评估，基于梯度敏感度的方法（Taylor 展开），基于特征图重建误差的方法。
- 典型方法举例：Filter Pruning via Geometric Median, HRank 等。

**段落2 — 非结构化剪枝与稀疏训练**
- 非结构化剪枝在权重级别置零，可达到更高稀疏率但需要专用硬件/库支持。
- 经典方法：基于幅度的剪枝（Magnitude Pruning）、Lottery Ticket Hypothesis、SNIP。
- 稀疏训练：RigL 等方法在训练过程中动态调整稀疏模式，允许稀疏结构随训练演化。本文第四章使用 RigL 作为在线剪枝方法。

**段落3 — 数据侧与模型侧"选择"的对照与不足**
- 从"冗余消除"的统一视角看：coreset 选择消除数据冗余（识别对训练贡献小的样本），模型剪枝消除参数冗余（识别对推理贡献小的权重）。二者本质上是对训练流程中不同维度冗余的"选择"。
- 现有工作将这两类选择完全独立进行：先做数据选择再在选出的子集上训练完整模型，或先训练完整模型再做剪枝。流水线式处理忽视了二者的潜在耦合。
- 这一不足正是第四章 SWaST 要解决的核心问题：揭示数据冗余与参数冗余的双向耦合，提出协同优化。

## 2.3 数据选择中的优化方法

**段落1 — 双层优化的基本框架**
- 双层优化的一般形式：外层目标优化上层变量（如数据选择策略），内层目标求解下层变量（如模型参数），内层的最优解依赖于外层决策。
- 在机器学习中的典型应用：超参数优化（外层=验证损失，内层=训练损失）、神经架构搜索（DARTS：外层搜索架构，内层训练权重）、元学习（外层跨任务泛化，内层任务内适应）。

**段落2 — 主要求解策略**
- 基于隐式微分（Implicit Differentiation）：利用内层最优性条件计算外层梯度，需要 Hessian 向量积，计算代价高。
- 基于迭代微分（Iterative Differentiation）：对内层训练过程做截断反向传播（truncated backpropagation），计算与截断步数成正比。
- 基于有限差分（Finite-Difference）：用扰动估计梯度，避免显式微分，但方差较大。
- 基于惩罚的一阶方法：将内层约束转化为外层目标的惩罚项，消除双层结构，允许联合优化。代表性工作包括 penalty-based bilevel reformulation。

**段落3 — 双层优化在数据选择中的应用与不足**
- 数据选择天然适合双层优化建模：外层优化数据选择策略（使验证性能最优），内层在选定子集上训练模型。
- 已有工作：数据重加权（learning to reweight）、coreset 选择（PBCS）等。
- 共性假设：这些工作���么假设内层训练到收敛，要么不显式考虑计算预算约束。当内层因预算限制无法收敛时，标准双层优化的理论基础（如内层最优性条件）不再成立。
- 这一不足直接引出第三章：CADS 通过惩罚松弛和一维损失近似来处理内层非收敛情况。

## 2.4 分布外检测（OOD Detection）

**段落1 — 问题定义与分类体系**
- 问题设定：给定在分布内（In-Distribution, ID）数据上训练的分类器 $f$，目标是设计评分函数 $S(x; f)$，使得 ID 样本得分高于 OOD 样本（或反之），通过阈值实现二分判别。
- 方法分类：(1) 基于分类器输出的方法（利用 softmax 概率、logits 或能量函数）；(2) 基于密度估计的方法（在特征空间建模 ID 数据分布）；(3) 基于距离的方法（计算测试样本与 ID 数据在特征空间的距离）。
- 本文聚焦后处理方法（post-hoc methods）：不修改训练过程，仅在推理时利用已训练模型的输出设计评分函数。

**段落2 — 基于输出层的后处理方法**
- MSP（Maximum Softmax Probability）：最简单的基线，以最大 softmax 概率作为 ID 置信度。简单有效但区分能力有限。
- Energy Score：用 LogSumExp 替代 softmax，理论上与输入的对数似然相关，比 MSP 更有区分力。
- ODIN：通过温度缩放和输入扰动增强 softmax 概率的区分度。
- 这些方法的共同点：均利用网络最终输出（logits 或 softmax），依赖的信息来源单一。

**段落3 — 基于中间特征的方法**
- 特征激活操作方法：ReAct 截断异常大的激活值，DICE 通过稀疏化权重提升 OOD 检测灵敏度，ASH 在特征层做激活整形。这些方法通过干预中间特征改善最终评分。
- 特征空间距离方法：Mahalanobis Distance 在各层特征空间计算到类中心的马氏距离，KNN 用 K 近邻距离衡量与 ID 数据的接近程度。
- 共同点：利用了比最终输出更丰富的中间表示，但大多数方法仍在全局平均池化（Global Average Pooling）之后操作。

**段落4 — 共性不足：池化前信息的缺失**
- 全局平均池化将 $C \times H \times W$ 的特征图压缩为 $C$ 维向量，这一操作丢失了每个通道内的空间分布信息。
- 现有后处理方法（无论是基于输出层还是基于中间特征）主要在池化后的特征上工作，未利用池化前通道内的激活分布模式。
- 这一信息空白正是第五章 NAP 的切入点：NAP 发现 ID 样本和 OOD 样本在池化前的通道内激活模式存在显著差异（激活先验），这一先验可作为独立于现有方法的新检测信号。
