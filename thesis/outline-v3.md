# 博士论文全文骨架（v3 — 双主线视角）

## 论文标题（暂定）

数据选择驱动的深度学习效率与可靠性优化研究

Data Selection-Driven Efficiency and Reliability Optimization in Deep Learning

## 双主线设计

本论文采用两条互补的叙事主线，从不同视角组织同一组研究工作：

| | 主线 A：阶段视角 | 主线 B：交互变量视角 |
|---|---|---|
| **核心问题** | 在深度学习的不同阶段，数据选择扮演什么角色？ | 数据选择与哪些关键因素存在交互，交互如何影响选择策略？ |
| **第三章** | 训练/微调阶段的数据选择 | 数据选择 × 计算资源约束 |
| **第四章** | 训练阶段的数据与模型联合选择 | 数据选择 × 模型容量约束 |
| **第五章** | 推理阶段的数据相关性判断 | 数据选择 × 推理时分布偏移 |
| **串联逻辑** | 训练侧（第三、四章）→ 推理侧（第五章） | 逐步扩展交互维度：计算资源 → 模型容量 → 分布偏移 |
| **优势** | 直觉清晰，读者容易把握全局脉络 | 揭示工作之间的深层统一性，论证更有学术深度 |
两条主线不矛盾——主线 A 回答"在哪里做数据选择"，主线 B 回答"数据选择与什么交互"。在论文写作中，主线 A 作为表层叙事引导读者跟随流程推进，主线 B 在关键节点（章节引言、过渡段落、总结）提供深层解释。

## 统一主线

数据选择是提升深度学习效率与可靠性的关键驱动力。从**阶段**的角度看，数据选择贯穿训练、微调与推理的不同环节；从**交互**的角度看，数据选择并非孤立的决策——它的最优策略取决于与之交互的关键因素。

本论文围绕数据选择展开三项研究。在训练/微调阶段，**计算资源约束**决定了最优数据子集的组成（第三章）；同样在训练阶段，**模型容量约束**与数据选择存在深层耦合，需要联合优化（第四章）；在推理阶段，**推理时分布偏移**要求判断输入数据是否仍处于训练数据选择所界定的有效域内（第五章）。

## 章节间过渡逻辑

### 第一章 → 第二章
绪论从阶段和交互两个角度提出了数据选择的三个子问题。在展开具体工作之前，第二章梳理各领域已有研究，为后续三章提供技术背景。

### 第二章 → 第三章
相关工作表明，现有训练数据选择方法普遍忽视计算预算这一关键约束，导致无算法能跨预算一致领先。第三章从训练阶段出发，研究数据选择与计算资源约束的交互，将计算预算作为显式变量纳入数据选择框架。

### 第三章 → 第四章
从阶段视角看，第三章和第四章同属训练侧的数据选择问题，第三章聚焦于数据子集的选择，第四章则将选择范围从数据扩展到模型。从交互视角看，第三章揭示了数据选择与计算预算的交互，第四章进一步追问：当模型容量本身也是变量时，数据选择与参数冗余之间存在怎样的耦合？

### 第四章 → 第五章
从阶段视角看，前两章聚焦训练侧，第五章将数据选择延伸至推理阶段。从交互视角看，前两章研究的是训练中的资源约束（计算预算、模型容量）与数据选择的交互，第五章转向另一维度——推理时分布偏移：测试数据可能不再属于训练时所选定的数据分布，需要判断输入是否处于模型有效数据域内。

### 第五章 → 第六章
三章工作完成了对数据选择问题的双重覆盖：阶段上从训练侧延伸到推理侧，交互维度上从计算资源到模型容量再到推理时分布偏移。第六章从这两个视角统一回顾，讨论局限性和未来方向。
