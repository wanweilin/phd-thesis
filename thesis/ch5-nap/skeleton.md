# 第五章 推理阶段的数据相关性判断 — 章节骨架

**阶段视角**：推理阶段的数据选择——模型部署后，判断输入样本是否处于模型的"有效数据域"内。
**交互视角**：数据选择 × 推理时分布偏移——训练阶段的数据选择界定了模型的有效数据域，当推理时输入分布发生偏移，需要判断输入是否仍处于该有效域内。

---

## 5.1 章节��言：推理时分布偏移下的数据相关性判断

**段落1 — 从训练侧到推理侧的转换**
前两章解决了训练阶段的数据选择问题。模型部署后，面临一个新的数据相关性判断问题：输入样本是否处于训练数据选择所界定的有效域内？

**段落2 — OOD 检测作为推理时的数据选择**
分布外检测（OOD Detection）本质上是在推理时对输入数据的"选择"——判断输入是否属于模型应当处理的数据范围。这与训练时的数据选择形成呼应：训练选择界定有效域，推理检测守护有效域。

**段落3 — 现有方法的局限与本章目标**
现有后处理 OOD 检测方法主要利用全局池化后的特征，丢失了池化前通道内的分布信息。本章提出神经激活先验（NAP），利用这一被忽视的信息源。

## 5.3 神经激活先验（Neural Activation Prior）

**段落1 — 核心观察**
在全连接层前的特征图（$C \times H \times W$）中，ID 样本在各通道内倾向于激活少量神经元且激活值较大，而 OOD 样本的激活更分散且值较小。

**段落2 — 两个统计量**
定义通道内的最大激活值 $\text{Max}(A_j)$ 和均值激活 $\text{Mean}(A_j)$，分析它们在 ID 与 OOD 样本间的分布差异。

**段落3 — 直觉解释**
ID 样本触发了与训练数据匹配的特征检测器，产生尖锐激活；OOD 样本无法精准匹配，激活更弥散。这一先验不依赖任何额外训练，是模型在 ID 数据上训练的自然副产品。

**段落4 — 向 Transformer 的扩展**
在 Vision Transformer 中，NAP 体现为 cls token 对其他 token 的注意力分布——ID 样本的注意力更集中。

---

## 5.4 方法：基于 NAP 的评分函数设计

### 5.4.1 CNN 评分函数

**段落1 — 评分函数定义**
基于信噪比思想，定义 $S_{\text{NAP}}(x) = \frac{1}{C}\sum_{j=1}^{C}(\text{Max}(A_j)/(\text{Mean}(A_j)+\epsilon))^2$。Max 作为信号强度，Mean 作为噪声水平。

**段落2 — 设计动机**
解释为何采用 SNR 形式而非直接使用 Max：Mean 的归一化消除了不同通道间激活尺度的差异，使评分在跨通道聚合时更鲁棒。

### 5.4.2 Transformer 扩展

**段落1 — 评分函数**
对于 ViT，NAP 评分简化为最终 Transformer 块中 cls token 注意力权重的最大值。

**段落2 — 简化的原因**
注意力权重归一化后均值为常数 $1/(l+1)$，因此 SNR 退化为 Max。

### 5.4.3 与现有方法的组合

**段落1 — 组合策略**
NAP 与现有方法通过加权几何均值组合：$S_{\text{NAP-E}} = -E(x)^w \cdot S_{\text{NAP}}(x)^{1-w}$。

**段落2 — 正交互补性**
NAP 利用池化前的通道内信息，现有方法利用池化后的全局信息，二者在信息来源上正交，组合后互补增强。
